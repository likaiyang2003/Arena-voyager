## 3. 代码风格指南 (Code Style Guidelines)

### JavaScript (`content.js`)
- **模块化**: 所有代码必须包裹在 **IIFE** (立即执行函数表达式) 中：
  ```javascript
  (function() {
      'use strict';
      // ... 代码 ...
  })();
  ```
- **缩进**: 使用 **4 个空格** (Not 2 spaces)。
- **引号**: 优先使用 **单引号** `'`，HTML 模板字符串使用反引号 `` ` ``。
- **命名规范**:
  - 常量: `UPPER_SNAKE_CASE` (例如 `SITE_CONFIG`, `ICONS`)。
  - 变量/函数: `camelCase` (例如 `createUI`, `toggleState`)。
  - DOM 元素变量: 通常保持简洁，或包含 `El` 后缀 (例如 `tooltipEl`)。
- **DOM 操作**:
  - 优先使用 `document.createElement` 和 `classList` 操作。
  - 对于复杂的 HTML 结构，可以使用模板字符串赋值给 `innerHTML`。
- **错误处理**:
  - 关键逻辑 (如解析 DOM) 应包含 `try...catch` 或空值检查 (`?.`)，防止因网站改版导致脚本崩溃。
  - `console.log` 仅用于关键生命周期调试，发布时应精简。

### CSS (`style.css`)
- **缩进**: 使用 **4 个空格**。
- **选择器**: 使用具体的类名 (例如 `.clean-header`)，避免使用过于宽泛的标签选择器，防止污染宿主网页样式。
- **颜色**: 使用 CSS 变量 (如 `var(--c-bg)`) 以支持主题切换。
- **Z-Index**: 确保侧边栏层级高于网页内容 (通常 `z-index: 9999`)。

### 提交/修改规则 (Modification Rules)
- **Manifest**: 修改 `manifest.json` 时需谨慎，确保 `permissions` 和 `matches` 最小化且准确。
- **兼容性**: 考虑到目标网站 (LMArena, Gemini) 可能会更新 DOM 结构，编写选择器时应尽量健壮 (例如检查多个可能的 class 或 ID)。
- **注释**: 关键逻辑块 (如 "UI 配置", "核心构建") 请保留中文注释标题，以便通过 grep 快速定位。
